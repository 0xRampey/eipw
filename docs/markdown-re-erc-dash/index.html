markdown-re-erc-dash

Other ERCs are referenced using ERC-X, not ERCX or ERC X.

Examples

error[markdown-re-erc-dash]: proposals must be referenced with the form `ERC-N` (not `ERCN` or `ERC N`)
  --> input.md
   |
27 | These are the backwards compatibility concerns for the ERC1234.
   |
   = info: the pattern in question: `(?i)erc[\s]*[0-9]+`

Explanation

Reports common mistakes when referencing other proposals, like omitting the hyphen (ERC1234), or replacing it with a space (ERC 1234).

For consistency, all proposals should be referenced in the same way.

Interfaces and contract names should be enclosed in backticks to avoid this error: `ERC1234`.
